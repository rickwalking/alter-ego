## What was built

Provider failures during synchronous `POST workflow/start` (observed live
2026-07-17: OpenCode Go 429 `GoUsageLimitError` → generic 500) now surface as
structured retryable errors:

- New `application/services/carousel/provider_errors.py`:
  `classify_provider_error(exc)` maps `openai.RateLimitError` /
  `anthropic.RateLimitError` → `provider_rate_limited`, other
  `openai.APIError` / `anthropic.APIError` (incl. connection errors) →
  `provider_unavailable`, anything else → `None` (caller re-raises).
- Route (`editorial_workflow.py` start handler): a scoped `except Exception`
  that classifies; unclassified re-raises unchanged. Classified: logs
  `workflow_start_provider_error` (project_id, detail, error) and raises
  HTTP 429 (`provider_rate_limited`) or 503 (`provider_unavailable`).
- Constants `ERR_PROVIDER_RATE_LIMITED` / `ERR_PROVIDER_UNAVAILABLE` in
  `domain/constants/carousel_workflow.py`.
