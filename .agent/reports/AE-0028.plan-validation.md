## Resolved Blocking Gaps

### 1. Canonical source of truth

Resolved by making
`skills/runtime/carousel-pipeline/contracts/hero_lower_third_v1.yaml` canonical
for rule IDs and values. Shared documents explain those IDs, typed Python loads
and enforces them, and prompt policy fragments are generated from the typed
object. CI compares repository files, packaged files, policy objects, prompt
context, documentation references, and ADR paths.

### 2. Artifact transaction

Resolved with an exact project-root layout:

```text
assets/
.staging/<operation_id>/
versions/<artifact_version>/
current.json
```

Artifact versions are deterministic SHA-256 values. Staging is validated and
atomically renamed before database compare-and-swap activation. Routes use the
database artifact version, so failed DB activation leaves an unserved orphan
and failed `current.json` replacement remains repairable. Retention, cleanup,
legacy fallback, and rollback are defined.

### 3. Creator asset security

Resolved by rejecting arbitrary URLs and client paths. Owner/admin multipart
uploads become managed content-addressed WebP assets after MIME, magic-byte,
decoder, size, pixel, animation, decompression, EXIF, and path-containment
checks. Existing external URLs are reported but never fetched.

### 4. Structured slide schema

Resolved with five `extra="forbid"` Pydantic union variants:

- `IntroSlideCopy`
- `SummarySlideCopy`
- `ContentSlideCopy`
- `ClosingSlideCopy`
- `CtaSlideCopy`

Required, optional, and forbidden fields are defined. PT and EN must share the
discriminator, content kind, selected structure, and item count. Existing
structured keys remain available through a one-release compatibility adapter.

### 5. Geometry

Resolved with standard and HD dimensions, named DOM selectors, slide-type copy
start ratios, exact containment and overflow predicates, footer gaps,
tolerances, near-limit warnings, rendered-line measurement, required-font
checks, and image decode checks.

### 6. Timeouts and concurrency

Resolved with operation-specific timeout defaults, one bounded repair attempt,
export retry rules, stable timeout errors, artifact build records, deterministic
idempotency, two compare-and-swap boundaries, race behavior, cancellation
cleanup, and worker-restart recovery.

### 7. Gherkin

Resolved with architecture-level scenarios for:

- valid bilingual review,
- repair success and repair failure,
- translation shape mismatch,
- overflow and image decode failure,
- invalid creator avatar,
- concurrent refinement,
- pre-promotion failure,
- post-rename CAS failure,
- restart recovery,
- legacy reads,
- unauthorized branding,
- URL rejection,
- slash-only delivery skills.

Each implementation task must cite the scenarios it implements.
