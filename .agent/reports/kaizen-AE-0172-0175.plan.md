## Proposals (for approval — NOT yet created)

### P1 — Fix the husky/lint-staged pre-commit hook for the `frontend/` layout  [ratchet: UP] — T2

- **Failure class:** #1. Root cause: `npx lint-staged` runs at repo root where
  neither a lint-staged config nor `eslint`/`prettier` resolve; the config lives
  in `frontend/package.json`.
- **Enforcement (exact files):** `.husky/pre-commit` — route frontend-staged files
  through the frontend toolchain, e.g. a root lint-staged config
  (`.lintstagedrc.json`) mapping `frontend/**/*.{ts,tsx}` to a command that runs
  in `frontend/` (`cd frontend && eslint --fix` / `prettier --write`), or split
  the hook to `cd frontend && npx lint-staged` for the frontend subtree (and a
  backend equivalent). Verify it works from repo root AND inside a git worktree.
- **AC:** committing a deliberately-unformatted staged `.ts` file from the repo
  root auto-fixes it via the hook (no `--no-verify` needed); the hook no longer
  ENOENTs; documented so contributors stop reaching for `--no-verify`.
- **Eliminates:** the recurring `--no-verify` bypass; restores the local
  format/lint safety net (ratchet UP — re-enables a check that is currently dead).

### P2 — Standardize diff-base resolution across diff-scoped gates  [ratchet: UP] — T1

- **Failure class:** #2. Root cause: each diff-scoped gate hand-rolls
  `BASE...HEAD` with no fallback, so a missing merge base silently degrades it.
- **Enforcement (exact files):** extract a shared `resolve-diff-base` /
  `changed-files` helper (the dead-code pattern: try `BASE...HEAD`, fall back to
  two-ref `BASE HEAD`, then advisory) and use it in `scripts/ci/check-integrity.sh`,
  `scripts/ci/eslint-changed.mjs` (`lint:changed`), and
  `frontend/scripts/check-dead-code.mjs`.
- **AC:** a stacked branch with no merge base still computes the correct changed
  set (proven by a test); no diff-scoped gate silently no-ops.
- **Eliminates:** silent gate degradation on stacked/diverged refs.

### P3 — Dead-FILE advisory (non-blocking)  [ratchet: HOLD] — T1 (optional)

- **Failure class:** #3. A non-blocking `frontend / Dead files (advisory)` report
  (knip file scope) — like `frontend / Duplication (tests, advisory)` — surfacing
  the ~117 unused files without a blocking baseline, so the signal is visible
  without the noise of a blocking gate. Pairs with the AE-0172 export gate.
- **AC:** advisory job reports unused files, never blocks; documented as advisory.
