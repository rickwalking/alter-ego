### What changed
`workflow_workers.py` logged `logger.exception("workflow_workers_error")` with no
`exc_info` bound, so the global `structlog.processors.format_exc_info` had nothing
to render and prod emitted the event with no `exception` field every 60s. Fixed
the call site to `logger.exception("workflow_workers_error", exc_info=True)`,
matching the working call sites (e.g. `carousel/nodes/images.py:258`). Global
structlog config was left unchanged (it was already correct).

### Underlying recurring failure (diagnosis)
Not reproducible from code without a live prod DB. Strongest suspect: **prod DB
schema drift**. Prod is bootstrapped via `create_all` with no Alembic, so when a
column is added to `CarouselProjectModel` in code (recent additions include
`is_public`, `workflow_status`, `presentation_policy_*`, `artifact_version`,
`slide_layout_strategy`) but is missing in prod, the worker's
`select(CarouselProjectModel)` queries (alert + new auto-reject) raise
`UndefinedColumn` on every tick. The now-emitted traceback will confirm/refute
this in prod so a definitive fix (Alembic baseline / column add) can follow.
