## Guiding principle

This is **not** a fresh design. It carries decisions already ratified in prior refinements
(AE-0284 status badges, AE-0272..0277 responsive epic, AE-0282 visual verification). The developer
copies existing patterns; they invent nothing.

## Decisions to carry (source-of-truth references)

1. **Token map** (canonical constants `frontend/src/constants/neon.ts`):
   - `var(--color-primary)` (stale purple `oklch(.62 .214 278)`, globals.css:97) ‚Üí neon cyan
     `NEON_CYAN #00d4ff` / `text-neon-cyan` / `var(--color-neon-cyan)`.
   - `var(--color-background)`/`--color-bg-card` + `--color-border` ‚Üí `BG_CARD #0d1324` +
     `NEON_CARD_BORDER rgba(255,255,255,.06)`.
   - undefined `var(--color-text)` ‚Üí `var(--color-text-primary)` (`TEXT rgba(255,255,255,.88)`);
     keep defined `--color-text-muted`.
   - ad-hoc `destructive`/`warning` callouts ‚Üí `NeonAlert` (`components/molecules/neon-alert.tsx`)
     / `NeonBadge` variants.
2. **Semantic status-badge map = AE-0284, reuse as-is** ‚Äî route every status the panel renders
   through `resolveWorkflowStatusVisual` (`workspace/workflow-status.ts`) + `WorkflowStatusBadge`,
   NOT a hand-styled span. Locked decisions: `in_progress` = cyan **pulse dot**; `awaiting_human` =
   magenta; **readyToPublish = teal (NOT green)**; published/completed = green; rejected/failed =
   red; status conveyed by **label + colour + dot, never colour alone**.
3. **Accessibility (AE-0284 a11y MUST)** ‚Äî pulse dot `animate-pulse motion-reduce:animate-none`
   (static under `prefers-reduced-motion`); label-override badges compose `aria-label` =
   `"<label>, <status>"`; `role="status"` (polite) ONLY on the badge that *is* the status.
4. **Responsive (AE-0272..0277, gated by `lint:responsive-dashboard`)** ‚Äî Tailwind utilities only,
   no hand `@media`, no layout-critical inline `style`; two-pane grids use
   `md:grid-cols-[minmax(0,1fr)_360px]`; touch targets ‚â•44px (`min-h-11`); no horizontal overflow
   at 360/768/1024px; neon identity preserved.
5. **i18n** ‚Äî fix hardcoded `Slide {n}` (`slide-progress-grid.tsx:41`) via `create.*` keys in en+pt;
   reuse existing `create.status.*` labels (en.json:310-320); no `text-transform` on raw enum.

## Non-Goals / guardrails

- Visual-token migration only ‚Äî no change to phase computation, stalled-timer, slide counts, data
  contract, or DOM structure/semantics. No new tokens. Do not restyle already-migrated siblings.

## Status domain (disambiguation)

This panel renders the **carousel workflow** status enum (pending/in_progress/awaiting_human/
approved/published/rejected/failed) via `resolveWorkflowStatusVisual` ‚Äî a **different domain** from
the `blog_posts` status of AE-0295/0296. The two must not be conflated. Enforce this
**structurally, not by review discipline**: type `resolveWorkflowStatusVisual`'s map as a
`Record<WorkflowStatus, ‚Ä¶>` (or add a unit test asserting its keys === the workflow-status enum
values) so a future status addition fails the build rather than silently hitting the unknown‚Üícyan
fallback (the same failure class AE-0295 fixes). Additionally, make the **runtime** `unknown`
branch **throw in dev/test** (not silently cyan), so a runtime-unknown status arriving from a
loose-typed path (e.g. a deserialized API string bypassing the typed call site) is observable, not
gracefully hidden ‚Äî cyan stays only as a last-resort prod guard. Name the single status-bearing
component in the six files (it gets `role="status"`; siblings are decorative).

## Testing (AE-0153 pure-refactor path)

- Unit/snapshot tests for the six components: assert they render `WorkflowStatusBadge` (not raw
  spans) and carry no `--color-primary`/`--color-text` references (a grep-style assertion or
  snapshot). Behavior assertions unchanged.
- **Visual verification (AE-0282)**: before/after screenshot at 360/768/1024 in Test Evidence.
- **A11y assertion**: axe/playwright check on the migrated panel asserting **exactly one**
  `role="status"` (polite) live region for the status, and the pulse dot honors
  `motion-reduce:animate-none`. Guards against SR-spam (over-applied `role=status`) or a missing
  live region.
- Gates: `frontend:lint` (Tailwind theme tree-shake AE-0279 + `lint:responsive-dashboard` AE-0277),
  typecheck. QA signs off the no-`.feature` classification.
