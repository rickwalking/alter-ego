# AE-0248 — Dev Summary

**Shared Deep Agents harness (checkpointer / interrupts / builder / middleware).** T3.
Branch `feat/agent-harness-ae0248`. Delivered in committed green increments.

## What shipped

### Increment 1 — checkpointer provider (commit `4324fd8f`)
Created `agents/harness/` with `checkpointer.py` (`build_checkpointer` +
`CheckpointerProvider` Protocol, split into `_build_postgres_saver` / `_build_sqlite_saver`
to clear PLR0911 without a suppression). Moved `_build_checkpointer` out of
`bootstrap/app_factory.py` (now imports it). Behavior-preserving.

### Increment 2 — generic interrupt helper (commit `eba6f309`)
Extracted the carousel-agnostic snapshot scan `iter_interrupt_values` to
`harness/interrupts.py`. The carousel-coupled `_merge_interrupt_review_payload`
**stayed in the engine** (it knows `CarouselWorkflowState`): importing
`rag_backend.application` into the harness would add a NEW `agents → application`
edge, which the import-linter forbids (down-only AE-0082 baseline). The engine's merge
now calls the harness helper.

### Increment 3 — builder + presets + chat wiring (commits `d5ed91da`, `fe256f95`, `f28e0d91`)
- `config.py` — `DeepAgentConfig` frozen dataclass (typed inputs; keeps the builder ≤3 args).
- `builder.py` — `build_deep_agent(config)` wrapping `deepagents.create_deep_agent`;
  routes `AGENT_KIND_CHAT` configs through `assert_no_chat_checkpointer` (single
  enforcement point for ADR-0013 / AE-0247).
- `middleware.py` — `build_summarization_middleware` (trigger 40 / keep 20 messages) +
  `build_human_in_the_loop_middleware` presets.
- `store.py` (`build_in_memory_store`), `memory.py` (`resolve_memory_paths`, drops missing).
- Both chat agents (`rag_agent.py`, `alter_ego_agent.py`) now build via the harness with
  `SummarizationMiddleware` and **no checkpointer**.
- `f28e0d91` removed a net-new `# type: ignore[arg-type]` from the builder test that the
  anti-gaming integrity gate flagged (explicit typed construction + real-preset sentinel).

## Deviation
- The carousel `_merge_interrupt_review_payload` was **not** moved to the harness (the
  handoff suggested moving "resume logic"). Rationale: it is coupled to `CarouselWorkflowState`
  and moving it would breach the `agents → application` import ratchet. Only the generic
  primitive was extracted. Documented in `harness/interrupts.py`.
- `store`/`memory` providers exist but are not yet wired to consumers (deferred to AE-0250);
  `HumanInTheLoopMiddleware` preset added but not wired (chat agents need only summarization).
