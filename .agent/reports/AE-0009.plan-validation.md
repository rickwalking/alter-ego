## Blocking Gap: Where does the error message come from?

**AC #3 says "displays the backend error message"** but the plan does not specify *which field* contains the error message when a phase fails.

From the validation research:
- **SSE error events**: Set `error` in the hook state (a string)
- **`workflowState`**: Has `phase_status`, `current_phase`, but **no `error_message` field** in `EditorialWorkflowState` type

If the error is only available via SSE (not persisted in workflow state), refreshing the page will lose it.

**Fix needed**: Add `error_message?: string` to `EditorialWorkflowState` type (backend) or use the hook's `error` state (frontend-only, ephemeral). The task should specify which.
