## What was built

Prod-400 root-cause fix (project f9e3e199, Langfuse traces 1a9d3a8c…/375358d6…):

1. **Cache-after-parse + eviction** — `SourceSynthesisAgent.extract_key_points`
   now caches the raw LLM response only AFTER `_parse_response` succeeds; a
   cache HIT that fails to parse is evicted (`AIResponseCache.delete`, new
   additive method) and falls through to a fresh LLM call, so caches poisoned
   by pre-fix deploys self-heal without restart.
2. **One JSON repair round-trip** — on first-parse failure the agent replays the
   malformed output to the model with the shared `JSON_REPAIR_PROMPT`
   (made public in `infrastructure/llm/json_utils.py`); repair success is cached,
   repair failure still raises `ValueError` (fail-closed).
3. **Observable workflow-start failure** — the route's `except ValueError` now
   logs `workflow_start_failed` (structlog, project_id + error) and returns the
   specific detail constant `ERR_RESEARCH_SYNTHESIS_FAILED`
   ("research_synthesis_failed") instead of the opaque access-control
   "Invalid request". Status stays 400.
