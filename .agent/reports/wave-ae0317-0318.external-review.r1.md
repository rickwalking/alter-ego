## Blind Review — AE-0317 / AE-0318

Verdict: **FAIL**

### BLOCKER
None (no security hole that ships unscraped SSRF or silent cache poisoning survived — the core hardening is correct).

### MAJOR
1. `editorial_workflow.py:198` — Route now maps **every** `ValueError` to `ERR_RESEARCH_SYNTHESIS_FAILED`. The spec scopes the new detail to *"synthesis failures"*; any other `ValueError` in `start_workflow` (validation, orchestrator bugs) is mislabeled as a research-synthesis failure. The route test (`test_start_synthesis_failure_is_logged...`) raises a generic `ValueError` and only asserts the new detail — so it **cannot fail** if a non-synthesis error is mislabeled; the scoping is untested and violated.
2. `source_synthesis_agent.py:_request_repair` — The repair `llm.ainvoke` is outside any `try`. A transport/provider error during the repair round-trip raises a non-`ValueError` → propagates past the route's `except ValueError` → **500 instead of 400**. Spec AC: "Repair failure → `ValueError` propagates… Status stays 400." New uncontrolled 500 path.
3. `agents/research_enrichment.py` — The ticket Scope and arch plan §4.1/§3 both mandate `application/services/carousel/research_enrichment.py` ("tool logic stays in `application/`; agent packages stay façades"). The diff lands the enrichment in `agents/`. Business logic (scrape/search/caps) now lives in the agent layer; per the arch plan also risks falling outside mutmut's `application/` scope. Layering violation.
4. Tests — The arch plan + AC require *"one consolidation test asserting BOTH RAG entry points and the HTTP route produce scraped content via the single service path (regression against re-introducing edge scraping)."* The diff updates `test_carousel_pipeline_consolidation_unit.py` to call `enrich_sources` **directly** — it never exercises `_start_editorial_workflow_for_rag` / `start_from_subagent`. If someone re-adds `_scrape_url_sources` at the edge, **no test fails**.
5. Tests — Observability ACs asserted nowhere or weakly: `research_url_blocked`, `research_url_cap_hit`, `research_search_failed` are spec-listed structlog events with **no test assertion** (the `test_unsafe_url_is_blocked...` and `test_search_failure_is_non_fatal` only assert behavior, not the log event). `research_url_scrape_failed` is asserted only in the legacy consolidation test. ACs explicitly require these to be logged; tests cannot catch their absence.

### MINOR
6. `source_synthesis_agent.py` — Spec says "Parse via the existing repair path (`extract_json_with_repair` or an equivalent seam…)". Implementation inlines a new `_request_repair` that (a) ignores `extract_json_with_repair`, (b) drops the `temperature=0.2` used by the battle-tested helper, (c) restructures message roles (raw as `AIMessage`, prompt as `HumanMessage` — arguably better, but a divergence from the proven path).
7. `agents/research_enrichment.py:_scrape_url_sources` — Unsafe URLs selected within the first 5 still **consume a scrape-budget slot** (job created, guard rejects, content preserved); a payload with unsafe URLs in positions 1–5 crowds out legitimate URLs 6–10. Spec says "at most 5 URL sources are scraped" — strictly satisfied, but the cap semantics are surprising.
8. `rag_agent.py:_build_subagents` — Gates researcher on `research_tool is not None AND carousel_tool_access is not None`. AC says "when carousel tool access is present." Reasonable (researcher needs the tool), but the `research_tool=None ∧ carousel_access=present` negative case is untested.
9. No test validates that `research_enrichment_enabled=false` actually wires `None` at `build_editorial_workflow_service` / `build_rag_agent`; only the `None`-tool no-op is unit-tested. The flag→wiring contract is untested.
10. `test_scrape_concurrency_is_bounded` uses `asyncio.sleep(0)` which may not force true overlap — assertion is `≤ 2`, so the test can pass even if the semaphore admitted only 1; weak (cannot robustly detect a regression to `Semaphore(3)` under all schedulers).

### NIT
11. `domain/models/research_enrichment.py` imports `ResearchTool` from `rag_backend.domain.protocols.carousel` while `agents/research_enrichment.py` and `api/dependencies/agents.py` import it from `rag_backend.domain.protocols` — inconsistent (works only if `protocols/__init__` re-exports).
12. `_parse_cached` catches `(ValueError, TypeError)`; `_parse_response` only raises `ValueError` — the `TypeError` branch is defensive but masks a possible contract change in `extract_json` silently.
13. `from None` is retained in the route despite now logging via `logger.exception` — the suppressed chain is in logs (fine), but consider dropping `from None` for cleaner tracebacks now that it's observable.
14. No dedicated test for the pure happy path "valid JSON cached → 2nd call makes 0 LLM calls"; only the repair-path caching is asserted.
