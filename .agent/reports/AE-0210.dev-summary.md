### What changed
The workflow worker now performs the auto-reject mandated by CLAUDE.md
("Auto-reject after timeout; never leave workflows stuck"), instead of only
emitting `stuck_workflow` warnings. Per worker tick, workflows that are not yet
`published`, sit in a non-terminal pending-like phase status (`pending` or
`awaiting_human`), and have `updated_at` older than the configured timeout are
transitioned to terminal `phase_status=rejected` / `status=failed`, get an
`error_message`, log `workflow_auto_rejected`, and emit the existing
`content.project.phase_changed` event through the transactional outbox.
`in_progress` workflows are excluded (may be actively resuming).

### Architecture note (import-boundary ratchet)
The application/worker layer must gain no net-new `application -> infrastructure`
import edge (arch-ratchet baseline = 61, 0 net-new allowed). So the ORM query +
transition live in a new **infrastructure** repository
`WorkflowTimeoutRepository` implementing a new **domain** protocol
`StuckWorkflowAutoRejector`. The worker depends only on the domain protocol plus
an injected `AutoRejectorFactory`; the composition root `bootstrap/app_factory.py`
(allowed to touch infrastructure) wires the concrete repository in.
