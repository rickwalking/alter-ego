# AE-0266 — Architecture Plan: declarative palette registry (single source of truth)

Status: Proposed · Tier: T2 · Area: backend (+ frontend sync) · Mode: plan (read-only)
Author: architect-skill · Date: 2026-06-23

## Problem

Theme/brand/palette config is expressed as **6+ parallel structures** that must be
edited in lockstep and silently desync:

| Concern | Today | Desync risk |
|---|---|---|
| color triple | `CAROUSEL_THEMES` / `BRAND_PALETTES` dicts | — |
| selectable enum | `CarouselTheme` StrEnum (hand-kept == keys) | enum/keys drift |
| light vs dark | `LIGHT_THEME_KEYS` frozenset | a light key missing here → dark scrim on light art |
| AUTO eligibility | `AUTO_ROTATION_THEME_KEYS` tuple | AUTO hands a light palette to a dark strategy (real AE-0264 bug) |
| brand detection | `BRAND_KEYWORDS` dict | keyword/palette separated |
| category detection | `THEME_CATEGORY_KEYWORDS` dict | — |
| image strategy | **two** maps: `image_provider_registry` + `image_prompt_package._STRATEGY_MAP` | drifted → light palette rendered with "neon glow" (real AE-0264 bug) |
| frontend | `create.ts` THEME_OPTIONS/THEME_LABEL_KEYS + en/pt i18n + zod enum | FE dropdown silently missed new themes (real AE-0264 bug) |

Three production bugs this session trace to this fragmentation. Adding **one** color
touches ~8 files. The 400-line file rule also forced splitting brand keywords away
from their palettes, which is the opposite of the desired cohesion.

## Root cause

There is **no single source of truth**. Each property of a palette lives in its own
parallel collection, so "a palette" is an emergent join across many dicts rather than
one declared thing.

## Options considered

**A. Builder pattern** — `Palette.dark("plasma_magenta", "#ec4899"…).with_keywords(…)`.
*Rejected as the core mechanism.* Builders shine for step-wise construction of complex
mutable objects; our palettes are flat, static value objects. A Builder adds ceremony
and a second representation without removing any parallel structure. (May be offered as
optional fluent sugar over the dataclass, but it is not the fix.)

**B. Registry of frozen descriptors (declarative table)** — one `PaletteDescriptor`
per palette/brand carrying *all* its properties; the legacy dicts/enum/sets are
**derived** from it. *Recommended.* Co-locates every property (incl. brand keywords +
palette), makes "add a palette = add one row", and the derivations can't desync because
they're computed, not hand-maintained.

**C. External declarative data file (YAML/JSON) + loader** — same as B but data lives
outside Python. *Deferred.* Useful later for FE codegen, but a typed in-code table keeps
mypy-strict guarantees and avoids a parse/validate layer now. We can *emit* JSON from B
for the frontend (see Sync) without sourcing from it.

## Recommended design (Option B)

### Domain value objects (`domain/models/` — entities only, allowed in domain)

```python
class PaletteMode(StrEnum):   LIGHT = "light"; DARK = "dark"
class PaletteKind(StrEnum):   CATEGORY = "category"; BRAND = "brand"; VARIANT = "variant"

@dataclass(frozen=True)
class Palette:                                   # the color triple
    primary: str; accent: str; background: str
    def as_theme_dict(self) -> dict[str, str]: ...  # legacy {"primary","accent","background"}

@dataclass(frozen=True)
class PaletteDescriptor:                          # ONE cohesive declaration
    key: str                                      # "plasma_magenta"
    palette: Palette
    mode: PaletteMode                             # light|dark  -> drives scrim + AUTO
    kind: PaletteKind                             # category|brand|variant
    image_style: str                              # the style it pairs with (neo_anime|flat_editorial|comic_neon)
    keywords: tuple[str, ...] = ()                # brand OR category detection terms
    auto_selectable: bool = True                  # eligible for AUTO rotation
    label_en: str = ""                            # single source for FE i18n
    label_pt: str = ""
```

Constraint kept by construction: a `LIGHT` descriptor whose `image_style` is a dark
strategy is rejected at registry-build time (a `__post_init__`/builder check) — the
AE-0264 light-on-dark class of bug becomes unrepresentable.

### The registry (one row per palette — brand keywords sit *with* their palette)

```python
PALETTE_REGISTRY: tuple[PaletteDescriptor, ...] = (
    PaletteDescriptor("cybersecurity", Palette("#ef4444","#00d4ff","#0a0e17"),
                      PaletteMode.DARK, PaletteKind.CATEGORY, IMAGE_STYLE_NEO_ANIME,
                      keywords=("security","exploit",...), label_en="Cybersecurity", label_pt="Cibersegurança"),
    PaletteDescriptor("anthropic", Palette("#ea580c","#22d3ee","#0a0b12"),
                      PaletteMode.DARK, PaletteKind.BRAND, IMAGE_STYLE_NEO_ANIME,
                      keywords=("anthropic","claude","opus",...), label_en="Anthropic", label_pt="Anthropic"),
    PaletteDescriptor("paper_editorial", Palette("#111827","#2563eb","#f7f5f0"),
                      PaletteMode.LIGHT, PaletteKind.VARIANT, IMAGE_STYLE_FLAT_EDITORIAL,
                      auto_selectable=False, label_en="Paper Editorial (light)", label_pt="Paper Editorial (claro)"),
    ...
)
```

### Derived views (computed once; keep the **exact existing public names + types**)

```python
CAROUSEL_THEMES        = {d.key: d.palette.as_theme_dict() for d in _non_brand}
BRAND_PALETTES         = {d.key: d.palette.as_theme_dict() for d in _brands}
BRAND_KEYWORDS         = {d.key: list(d.keywords) for d in _brands}
THEME_CATEGORY_KEYWORDS= {d.key: list(d.keywords) for d in _categories}
AUTO_ROTATION_THEME_KEYS = tuple(d.key for d in registry
                                 if d.auto_selectable and d.mode is DARK and d.kind is CATEGORY)
LIGHT_THEME_KEYS       = frozenset(d.key for d in registry if d.mode is PaletteMode.LIGHT)
```

`resolve_theme()` and every current import keep working unchanged — they consume the
derived constants, which are byte-identical to today's hand-written values.

### `CarouselTheme` enum — keep explicit, guard with a contract test

A functionally-generated `StrEnum` loses static members + IDE/mypy ergonomics. Keep the
enum hand-written but add the **drift-guard test** pattern already adopted this session
(AE-0264 `_STRATEGY_MAP` guard): assert `{t.value for t in CarouselTheme} - {"auto"}
== {d.key for d in PALETTE_REGISTRY}`. Adding a palette without its enum member fails CI.

### Kill the two-strategy-map drift (folds in here)

Introduce one `IMAGE_STRATEGY_REGISTRY: dict[(model,style), type[ImageStyleStrategy]]`
consumed by **both** `image_provider_registry` and `image_prompt_package`. Each
`PaletteDescriptor.image_style` is validated against it. This eliminates the AE-0264
drift bug at the structural level (replaces my point-fix drift-guard test).

## Frontend single-source-of-truth

The backend registry is canonical; FE stays in sync by **contract**, matching the
project's existing `frontend/schema-drift` gate philosophy:

1. Backend emits `docs/contracts/palettes.json` (built from the registry; a unit test
   asserts the file is regenerated — fail if stale).
2. Add `frontend` check `lint:palette-drift` asserting `create.ts` THEME_OPTIONS +
   THEME_LABEL_KEYS + zod `image_style` enum + the `(model,style)` combos all match
   `palettes.json`. (Mirror of the existing schema-drift checker.)
3. i18n labels come from `label_en`/`label_pt` in the contract → no hand-typed FE labels.

(Phase 3 optional: a runtime `GET /api/carousels/palettes` so the FE renders the catalog
dynamically and drops the generated list entirely.)

## Migration plan (incremental, each phase independently shippable, non-breaking)

- **Phase 1 (core, ~1 PR):** add `Palette`/`PaletteDescriptor`/`PALETTE_REGISTRY` in
  domain; rewrite `carousel_themes.py` constants as **derived** from the registry
  (same names/values/types). Add enum↔registry + light-implies-light-style guard tests.
  No consumer changes; resolve_theme untouched; all 13 palettes + 6 brands preserved.
- **Phase 2:** consolidate image-strategy selection into one registry used by both
  call sites; descriptors reference it; remove the duplicate map.
- **Phase 3:** emit `palettes.json`; add the FE `palette-drift` gate; drive FE
  options + i18n from labels.
- **Phase 4 (optional):** runtime palettes endpoint; FE dynamic load; derive enum.

## Testing

- Derivation parity: each derived constant equals a frozen snapshot of today's values
  (proves zero behavior change in Phase 1).
- Invariants (property/example): every descriptor's `image_style` ∈ strategy registry;
  every LIGHT descriptor pairs with a light strategy; AUTO pool excludes light/brand;
  enum == registry keys; `resolve_theme` brand/category/hash paths unchanged.
- FE: `palette-drift` contract test (Phase 3) + existing create-page i18n test.
- Mutation: the derivation + invariant helpers are business logic → mutmut target.

## Risks / rollback

- *Risk:* a derivation bug silently changes a palette. *Mitigation:* the frozen-snapshot
  parity test in Phase 1 is the gate; Phase 1 is a pure refactor behind stable names.
- *Risk:* enum can't be fully derived ergonomically. *Accepted:* keep explicit + guard test.
- *Rollback:* Phase 1 is self-contained; revert restores the literal constants (they're
  the same values). Later phases are additive.

## Clean-architecture fit

Domain holds the descriptors + value objects (entities only — no app imports). Derivation
is pure functions. No `Any` (frozen dataclasses + `StrEnum` + `Protocol` for the strategy
contract). Each `PaletteDescriptor` ≤ a few fields (max-3-arg rule met via the dataclass).
File size: one descriptor per row is compact; data table and derivation logic can be two
small modules, but **brand keywords stay co-located with their palette** (the row), which
is the explicit requirement.

## Decision Log (for the ticket)

- Builder pattern: **rejected** as core (static flat config); optional fluent sugar only.
- Source of truth: **typed in-code registry** (not YAML) to preserve mypy-strict; emit
  JSON for FE.
- Enum: **explicit + CI drift-guard** (not generated).
- Strategy maps: **consolidate to one** (kills the AE-0264 drift class).
- FE sync: **contract gate** mirroring existing schema-drift (Phase 3); runtime API later.
