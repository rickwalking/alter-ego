## Blocking Gaps

1. **AE-0039 AC1 and AC4 are mega-checklists.** They bundle multiple unrelated checks into single ACs. If E2E evidence fails, the QA report cannot pinpoint which sub-item failed without an internal checklist. Splitting them or adding a sub-checklist is required before release.
2. **AE-0039 Gherkin lacks failure paths.** No scenario covers: failed backfill (data corruption risk), failed regeneration (rollback failure), or mutation test failure blocking release.
3. **AE-0034 Gherkin lacks stale-version and extras-editing failure paths.** AC3 requires rejecting stale edits, but no Gherkin scenario covers this.
4. **AE-0039 does not explicitly reference ADR-005 mutation thresholds.** AC1 says "mutation targets ... SHALL all pass" but does not state the 70% business-logic threshold. Without the threshold, the pass/fail boundary is undefined.
5. **AE-0039 has 10 blocking dependencies (AE-0029 to AE-0038).** If any child ticket is delayed or its scope changes, E2E evidence collection is blocked. The ticket should include a risk AC for dependency drift.

## Suggested AC Additions

1. **AE-0039 — Add explicit mutation threshold AC:** "WHEN mutation testing runs on business logic modules THE mutation score SHALL be >= 70% per ADR-005 thresholds."
2. **AE-0039 — Add dependency readiness AC:** "WHEN any child ticket blocks E2E evidence collection THE release QA SHALL report the blocker and skip downstream gates until resolved."
3. **AE-0039 — Add rollback failure AC:** "WHEN explicit regeneration fails THE SYSTEM SHALL leave the prior artifact version intact and log the failure reason."
4. **AE-0034 — Add stale version rejection Gherkin scenario:** "Given the reviewer edits content after the backend version has changed, When submit is attempted, Then the UI rejects with a stale version error."
5. **AE-0034 — Clarify AC1 schema:** Add a sub-list or reference to the exact schema keys that constitute "structured extras" (e.g., `extra.headline`, `extra.cta`, `extra.icon`).

## Sign-off

All issues are resolvable with minor AC edits and Gherkin additions before the Developer Skill begins implementation.
