## Developer Completion Report
Ticket: AE-0208 — Provider-rate-limit-aware image generation (concurrency cap + retry-after)
Status: Dev Complete

### Acceptance Criteria Implemented
- [x] Concurrency capped to a config value; no uncapped `gather` fan-out for image calls.
  Each provider call now runs under an `asyncio.Semaphore` sized from the new
  `carousel_image_concurrency` setting (default 5 = the documented OpenAI org
  per-minute cap). The `asyncio.gather` fan-out still schedules all slides, but
  in-flight provider calls never exceed the cap.
- [x] 429 `retry-after` honored (backoff ≥ stated wait).
  `generate_with_retry_after` retries a single generation on HTTP 429, waiting
  the provider's `retry-after` header value plus a small margin, falling back to
  bounded exponential backoff when the header is absent. Attempt budget is
  `carousel_image_max_attempts` (default 5). Non-429 errors and the final 429
  re-raise unchanged so existing failure handling/records are preserved.
- [x] Seeded-429 test passes (the phase completes, does not abort).
