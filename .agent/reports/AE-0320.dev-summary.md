## What was built

Fix for the prod gate-approval self-deadlock (5/5 repro on 2026-07-18,
evidence `/root/ae-outline-wedge-evidence.txt` on the droplet):

1. **Lock-timeout-bounded run writers** — new
   `infrastructure/database/pg_lock_timeout.py` applies a transaction-scoped
   `SET LOCAL lock_timeout` (`RUN_WRITE_LOCK_TIMEOUT_MS = 2000`) on Postgres
   (dialect-gated no-op on SQLite, mirroring the AE-0316 advisory-lock
   pattern). Applied by `write_run_heartbeat` and the reaper's atomic flip —
   the two separate-session writers that queued behind the runner's own
   uncommitted row lock.
2. **Single-attempt stage beats** — `_beat_and_stage` (awaited INLINE by the
   run's main coroutine) now uses the new `write_run_heartbeat_once`
   (never raises, no retry loop, warning-logged). The 60s liveness loop keeps
   the retry variant. This removes the self-deadlock's await edge entirely.
3. **Reaper fail-fast** — a flip that hits the lock timeout logs
   `carousel_run_reap_blocked`, rolls back, and retries next tick instead of
   wedging the whole worker tick behind a live run's transaction.
4. **Phase-drift convergence** — the AE-0311 drift reconciler gains a second
   pass: a row left `failed` by a dead run whose checkpoint advanced and
   parked `awaiting_human` at a DIFFERENT phase is converged onto the
   checkpoint (phase + awaiting_human, lock_version + run_epoch bumped, run
   columns cleared, `carousel_phase_drift_converged` logged). Same-phase
   failures and mid-step checkpoints are untouched (recovery UI owns them).
   The raw-update allowlist gate + its lockstep survey test cover the new
   write site.
