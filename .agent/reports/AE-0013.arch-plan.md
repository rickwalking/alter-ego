# Architecture Plan: Carousel Render Polish & Visual QA Epic

**Epic:** AE-0013 to AE-0016  
**Status:** Plan  
**Source:** `.impeccable/critique/2026-06-08T01-11-06Z__carousel-191223a4-rendered-slides.md`  
**ADR required:** No. This is a deterministic rendering/template polish and QA scope. It does not change workflow orchestration, persistence contracts, auth, or LLM behavior.

---

## Problem

The carousel pipeline now produces valid PT/EN rendered slides, but the design critique found that the rendered output still fails at feed-preview scale:

- The creator avatar/watermark is barely visible on content slides.
- The closing slide avatar is too small and the website button overpowers Pedro's identity.
- Slide title/body text is readable at full resolution but conservative for social feed compression.
- Current page/total slide indicators are too small.
- PT slides still render `Deslize →`, but the requirement is `Swipe →` even for Portuguese.
- There is no visual QA gate for readable-but-too-small rendered slides.

The current design-health score is **24/40** with three P1 issues.

---

## Goal

Ship an art-direction polish pass that makes rendered carousel slides legible and brand-recognizable at social feed scale, while preserving the Neon Shell cyberpunk image system and current PT/EN generation flow.

---

## Non-Goals

- No new image generation provider changes.
- No changes to LangGraph workflow phase logic.
- No frontend editor redesign.
- No broad carousel content rewriting.
- No change to output dimensions (`1080x1350` standard, `2160x2700` HD export).

---

## Task AE-0013 — Swipe Copy Consistency

### Scope

Update all generated carousel preview/design paths so PT and EN both use `Swipe →`.

### Candidate Files

- `backend/src/rag_backend/application/services/carousel_template/strategies/hero_content.py`
- `backend/src/rag_backend/application/services/carousel_template/design.py`
- Existing strategy/design token tests under `backend/tests/unit/application/`

### Acceptance Criteria

1. **WHEN** a PT carousel slide renders the slide-2 swipe cue, **THEN** the rendered HTML uses `Swipe →`.
2. **WHEN** an EN carousel slide renders the slide-2 swipe cue, **THEN** the rendered HTML uses `Swipe →`.
3. **WHEN** PT design tokens are generated, **THEN** `layout.swipe_text` equals `Swipe →`.
4. **WHEN** EN design tokens are generated, **THEN** `layout.swipe_text` equals `Swipe →`.
5. **WHEN** the preview design endpoint is requested for PT, **THEN** the response does not contain `Deslize`.
6. **WHEN** existing language constants are used elsewhere, **THEN** no unrelated translation behavior changes.

### Tests

- Unit test `HeroContentStrategy.render(...)` for PT and EN.
- Unit test `generate_design_tokens(...)` for PT and EN swipe text.
- Focused ruff on changed files.

## Task AE-0014 — Typography, Progress, and Creator Chrome Scale

### Scope

Increase the visual scale of main copy and identity/progress chrome so slides survive UI thumbnails and social feed previews.

### Candidate Files

- `backend/src/rag_backend/application/services/carousel_template/css/base.py`
- `backend/src/rag_backend/application/services/carousel_template/css/slide_styles.py`
- `backend/src/rag_backend/infrastructure/external/playwright_export.py`
- Strategy/property tests for generated CSS and export overrides

### Design Targets

- Increase content/hero title and body font sizes by roughly 10-15%, guarded against PT overflow.
- Increase `.slide-number` and `.slide-hero-number` enough to be visible in contact sheets.
- Increase `.counter-label`, `.counter-dot`, and active counter width.
- Increase `.s1-swipe` and keep it visually tied to the footer utility row.
- Increase `.creator-watermark-avatar`, name, and handle by roughly 2x to 3x in exported output.

### Acceptance Criteria

1. **WHEN** a content slide renders, **THEN** title/body text is larger than the current baseline and remains inside the slide bounds.
2. **WHEN** a hero-content slide renders, **THEN** title/body text is larger than the current baseline and remains inside the slide bounds.
3. **WHEN** a slide renders a `0N / 07` label, **THEN** the label is visually larger than the current 11px base style.
4. **WHEN** slide-2 renders the swipe cue, **THEN** the cue is larger than the current 11px base style.
5. **WHEN** slide-2 renders the creator watermark, **THEN** the avatar, name, and handle are materially larger in both base CSS and Playwright export overrides.
6. **WHEN** the HD export override is applied, **THEN** watermark and progress sizes scale proportionally with the 2160px canvas.
7. **WHEN** PT slide text is longer than EN text, **THEN** text wraps without clipping or overlapping the footer/watermark.

### Tests

- Unit tests/assertions for CSS tokens or generated CSS snippets.
- Existing carousel template strategy tests.
- Render smoke test for PT and EN sample slides.
- Manual/automated contact-sheet inspection at 25-33% scale.

## Task AE-0015 — Closing Slide Brand Payoff

### Scope

Recompose the final slide so it works as a memorable Alter-Ego identity endpoint, not a sparse follow card.

### Candidate Files

- `backend/src/rag_backend/application/services/carousel_template/strategies/cta.py`
- `backend/src/rag_backend/application/services/carousel_template/css/slide_styles.py`
- `backend/src/rag_backend/infrastructure/external/playwright_export.py`
- `docs/guides/carousel-closing-slide-style.md`
- `backend/tests/unit/application/strategies/test_cta.py`

### Design Direction

- Make the avatar a primary element, not a tiny badge.
- Reduce dead vertical space above the identity stack.
- Balance avatar, name, handle, website, and CTA so the website button does not overpower Pedro's face/name.
- Update the guide so 72px is no longer treated as sufficient for the real export.

### Acceptance Criteria

1. **WHEN** the final slide renders, **THEN** the avatar is visually prominent and larger than the current 72px base style.
2. **WHEN** the final slide renders, **THEN** the page count remains readable above or near the identity stack.
3. **WHEN** the website button renders, **THEN** it does not visually dominate the avatar/name hierarchy.
4. **WHEN** the final slide renders in PT, **THEN** the CTA text remains in Portuguese and does not clip.
5. **WHEN** the final slide renders in EN, **THEN** the CTA text remains in English and does not clip.
6. **WHEN** `docs/guides/carousel-closing-slide-style.md` is read, **THEN** it reflects the updated avatar/hierarchy guidance.
7. **WHEN** the closing strategy is tested, **THEN** it still avoids using `images/slide_7.jpg`.

### Tests

- Unit tests for CTA strategy HTML.
- CSS/export override snippet assertions.
- PT/EN rendered slide visual check.
- `git diff --check`, ruff on changed files.

## Task AE-0016 — Render Visual QA Gate

### Scope

Add a focused QA path that catches regressions where image files are valid but the rendered slides are visually unsuitable.

### Candidate Locations

- A script under `scripts/` or `backend/scripts/` for local visual audit.
- Backend tests where deterministic checks are practical.
- `.agent/reports` QA checklist updates for manual visual review.

### Acceptance Criteria

1. **WHEN** the QA gate runs for a carousel project, **THEN** it verifies PT and EN rendered slide images exist for slides 1-7.
2. **WHEN** the QA gate validates rendered images, **THEN** each image has expected dimensions and non-zero file size.
3. **WHEN** the QA gate checks design preview JSON, **THEN** PT and EN expose seven rendered slide URLs.
4. **WHEN** the QA gate checks swipe copy, **THEN** PT and EN design preview JSON contain `Swipe →` and not `Deslize`.
5. **WHEN** contact sheets are generated, **THEN** PT and EN sheets are written to `/tmp` or a documented output path.
6. **WHEN** the QA gate cannot authenticate or the backend is down, **THEN** it fails with a clear actionable error.
7. **WHEN** the QA gate completes, **THEN** it prints a concise pass/fail summary suitable for PR review.

### Tests

- Unit tests for helper functions if the QA script has parsing/validation helpers.
- Manual run against carousel `191223a4-9499-4e66-84d6-e78bdee4e695` after AE-0013 to AE-0015.
- Documented command in QA report.

## Risks and Mitigations

| Risk | Severity | Mitigation |
|---|---:|---|
| Larger PT text clips or overlaps footer | Medium | Render PT and EN before merge; keep line-height and max-width guards |
| Larger watermark covers important image detail | Low | Keep watermark on slide 2 only; use stronger dark capsule, bounded width |
| Closing slide becomes too crowded | Medium | Treat avatar/name as primary, website as secondary CTA; inspect at full and contact-sheet scale |
| CSS changes affect non-HD preview differently than HD export | Medium | Update base CSS and both Playwright export override blocks together |
| QA gate becomes flaky due Docker/API availability | Medium | Keep deterministic checks separate from visual/manual steps; fail with clear environment message |
