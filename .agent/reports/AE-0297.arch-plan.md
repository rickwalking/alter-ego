# AE-0297 — Architecture Plan: public vs private blog-post read APIs

Ticket: AE-0297 · Tier: T2 · Type: Bugfix→Feature (contract change) · Author: architect-skill
Date: 2026-07-01
Decision source: user directive — "two blog post endpoints: one public (serves the homepage),
one private with more capabilities (shows more detail, incl. drafts)."

## Decision (resolves the AE-0297 open question)

**Option A, formalized as a two-endpoint split.** Add a dedicated **public**, unauthenticated
blog-post read surface that serves only `published` posts through a **lean** schema, and keep the
existing **private/admin** (`EditorUser`) surface that returns full detail including drafts. This
supersedes the "carousel-projection-only" reading (Option B).

Requires **ADR-0013** (public content contract changes: the public site will read the
`blog_posts` table, not only carousel projections). Draft: `docs/decisions/0013-public-private-blog-post-api.md`.

## Context (verified current state)

- Every `GET /api/blog-posts*` route is `EditorUser`-gated (`api/routes/blog_post.py:123,159`);
  there is **no** unauthenticated `blog_posts` read path.
- The public homepage + public blog listing read **carousel projections** today
  (`fetchCompletedProjects` → `GET /api/carousels?public=true`; detail → `GET /api/carousels/{id}/blog/{lang}` +
  `/design`), so a real `blog_posts` UUID 404s publicly (`(public)/blog/[id]/page.tsx:44`).
- `BlogPostStatus` (`domain/constants/blog_post.py:6-13`): `draft | under_review | approved |
  published | archived`. Only `published` is publicly visible. `status` is index-backed
  (`idx_blog_posts_status`), so a published-only query is cheap.
- Reusable public pattern already exists on carousels: `get_optional_user` (`dependencies/auth.py:144-166`)
  + an `assert_*_public` 404 gate (`carousels/carousel_access.py:19-23`), single fully-public read
  (`carousels/media.py:153-178`, no `user` dep).
- ADR-0011 / ADR-009: `blog_posts` row is the **single source of truth** (carousel→blog projection
  already writes into it). A public read model must **not** create a second source of truth — it is
  a read-only projection over the same row.

## API contracts

### New — public surface (NO auth dependency — role-blind by construction)
- `GET /api/public/blog-posts?limit=&offset=&tag=` → `PublicBlogPostListResponse`
  - Server-forced filter `status == published`; ordered by `published_at DESC`. Ignores/rejects any
    client-supplied status filter.
- `GET /api/public/blog-posts/{id_or_slug}` → `PublicBlogPostResponse`
  - `assert_blog_post_public(post)`: **uniform 404** for every non-published state
    (draft/under_review/approved/archived alike). No 403, no 410 — a uniform 404 leaks neither
    existence nor lifecycle (410-for-archived would reveal a resource once existed; rejected).
- **No `get_optional_user`** — route resolves no user identity (mirrors carousel media route
  `media.py:153-178`). **Two structural guards:** (1) `.feature` — authenticated editor gets
  byte-identical anon lean payload for a published post, 404 for their own draft; (2) test asserts
  the route handler's **dependency tree has no auth dep** (fails the build if `get_optional_user`/
  `EditorUser` is ever added) — closes the silent-regression path (payload equality alone wouldn't
  catch a telemetry/logging leak).
- **Anon server-fetch (verified):** `lib/server-fetch.ts` `validatedFetch` sends no headers/cookies/
  credentials — the SSR path hits the public route as a true anonymous client. Prod reachability via
  `API_BASE_URL` override in `getBaseUrl()`; integration test asserts no cookie/Authorization sent.
- **Rate-limit (committed) + cache:** committed per-IP rate limit (baseline 60 req/min → 429,
  `.feature`-asserted, tune threshold in impl) and `Cache-Control: no-store` (pinned; asserted on
  both list + detail). Any future CDN needs an ADR amendment + tested purge on unpublish.

Lean public schemas (new, in `api/schemas/blog_post.py`) — **explicit allow-list** (built by
field-by-field mapping, never `from_orm`):
- Public summary: `id, slug, title, excerpt, featured_image_url, published_at, meta_title,
  meta_description, keywords, canonical_url, origin, project_id`. (SEO is flat public-safe columns —
  there is **no** `seo_meta` JSONB. `origin`+`project_id` let the FE enrich carousel-origin posts.)
- Public detail: summary + `content` (rendered body).
- **Omitted in v1:** `view_count` (no write-on-read; see below), `distribution`, `sources`,
  `citations` (JSONB/list blobs not needed to render a page; avoid nested pass-through).
- **Never serialized:** `status`, `author_id`, `reviewer_id`, `editor_comments`, `version_history`,
  `ai_suggestions`, `ai_generation_metadata`, `lock_version`.
- **Recursive leak test:** serialize the payload and assert none of the excluded key names appear
  anywhere in the JSON (incl. nested objects) — closes the nested-JSONB leak (B3).

### Read-only + abuse controls (in this ticket, not deferred)
- Public GET performs **zero DB writes**. No `view_count` increment on read (would break the ADR
  read-only invariant + create an anon write/contention/scraping vector). Future view analytics =
  out-of-band Redis-Stream event, separate ticket.
- Add a basic **rate-limit** to the public routes and an explicit **`Cache-Control`** header
  (short `max-age`, or `no-store` pre-CDN) in this ticket. When a CDN lands, `unpublish` must purge
  the public cache for the id/slug (owned follow-up).

### Slug resolution — id-only in v1
`slug` is UNIQUE-enforced, but a two-arity `{id_or_slug}` lookup risks an id/slug namespace-collision
oracle (a token resolving to a *draft* by slug returns 404 while the same token resolving to a
*published* post by id returns 200 — defeating the uniform-404 property). **v1 resolves by `id`
only.** Slug resolution is a deferred follow-up whose **precondition must be verified against prod
data first**: query `blog_posts` for any UUID-shaped `slug` values (must be zero) before enabling
slug lookup, plus an explicit id-then-slug precedence + collision test. Do not treat the deferral as
"resolved" without that data check. Keeps the uniform-404 property intact in v1.

### Deploy sequencing (cross-ticket, load-bearing)
The AE-0296 DELETE **409 guard** and the **carousel-deletion → flip-orphan-to-draft** flow are
mutually load-bearing: shipping the 409 guard *before* the flip lands would create a window where
carousel-origin posts are un-deletable AND deleting their parent leaves a phantom published row.
**Order: the carousel-delete→flip-to-draft change lands before or atomically with the 409 guard.**
The flip is NOT yet implemented (`models/blog_post.py:28-32` is only `ON DELETE SET NULL`, the
opposite of a draft-flip) — so it is **in scope to build**, with an integration test: delete a
carousel project whose blog row is `published` → assert the row is `draft` (not public, not
orphaned) afterward. Owner: AE-0296 (with AE-0297).

### Fallback lean-invariant ownership
The carousel public feed is a **separate evolving endpoint**; its "stays lean" property must be
defended where it lives. AE-0297's fallback test asserts the intersection of the blog excluded-key
set against the carousel payload, but a **carousel-endpoint-owned recursive-leak test** is the real
guard against a future carousel PR widening the feed. Flag that as the carousel endpoint's own
invariant (own suite), not something AE-0297 can fully own.

### Existing — private/admin surface (unchanged contract, "more capabilities")
- `GET /api/blog-posts` (`EditorUser`) — full list, all statuses incl. draft, optional `status`
  filter, non-admin scoped to own author_id. Already satisfies "shows drafts / more detail."
- `GET /api/blog-posts/{id}` (`EditorUser` + `get_blog_post_for_read`) — full `BlogPostResponse`.
- No behavior change required here beyond confirming drafts remain visible to authors/admin.

## Data model

No schema/migration change. `status` + `published_at` already exist and are indexed. Public
queries are read-only projections. If `slug` lookups are supported, confirm `slug` uniqueness
(`models/blog_post.py`); if not unique/enforced, resolve by `id` only in v1 and defer slug.

## Frontend

- Public homepage feed + `(public)/blog` listing: add a public API client
  (`fetch('/api/public/blog-posts?...')`) and switch the homepage/blog listing to it (or run both:
  see Non-Goal below). New Zod schema `publicBlogPostListResponseSchema`.
- Public detail `(public)/blog/[id]/page.tsx`: **single canonical URL `/blog/{id}`**, branch by
  `origin`. `origin == carousel` → render via the existing carousel design path (design tokens);
  non-carousel → render from `PublicBlogPostResponse` with a **default public theme** (do not 404
  for missing design tokens). Enrichment for carousel-origin uses the returned `project_id` +
  existing `/design`. This yields one URL per post (no dual-feed divergence / SEO duplicate).
- **Prod reachability is a primary AC:** integration test that the server-side fetch resolves
  against the real internal hostname (`getBaseUrl()` / `API_BASE_URL` / `PRODUCTION_BACKEND_URL`,
  server-fetch.ts:15-25).
- **Fallback: list fail-open (no new exposure), detail fail-closed.** The **list** may fall back to
  the **existing** public carousel feed on error — rendered through its **own existing validated
  schema/component**, never merged into/widened past the lean schema, so it exposes nothing not
  already public today (it *is* today's homepage source). The **detail** page fails closed
  ("temporarily unavailable") — no silent shape switch. **The fallback path is inside the
  security-test surface, not argued by ancestry:** a test asserts the carousel-fallback feed payload
  contains **none** of the `blog_posts` excluded keys (author/reviewer/editor/AI-internal) — closing
  the "fallback sits outside the recursive-leak test" gap.

## Testing

- Backend: `.feature` (behavior-changing) — public list returns only published; public detail 404s
  for draft/under_review/approved/archived/unknown; lean schema **never** contains
  editor/AI/author internal fields (explicit assert on excluded keys — security regression test);
  private routes still return drafts to author/admin. Unit tests for `assert_blog_post_public`.
- Frontend: public listing renders published feed; detail renders a non-carousel published post
  with default theme; hidden/unpublished (AE-0296) → not-found; Zod parse of lean payload.
- Cross-layer (per cross-layer-validation-gap memory): a full FE→BE test that the example prod URL
  shape resolves.

## Security

- No-leak 404 for non-published (existence not disclosed).
- Lean schema is an **allow-list** of fields, not a deny-list, to prevent future field additions
  from silently leaking (build `PublicBlogPostResponse` by explicit field mapping, not
  `from_orm` of the full model). This is the single most important security control here.
- Public endpoints are unauthenticated by design; add basic rate-limit/caching consideration
  (public GET, `cache: no-store` currently — consider short CDN cache, non-blocking).

## Rollout / rollback

- Additive: new `/api/public/blog-posts*` routes + new schemas; no change to existing private
  routes or DB. Rollback = remove the public routes + revert the FE switch; private admin flow
  unaffected.
- Sequence after AE-0295 (listing no longer crashes) and coordinated with AE-0296 (hide/unpublish
  must flip a post out of `published` so the public filter drops it — verify archived/unpublished
  disappear publicly).

## Non-Goals

- Not merging the carousel-projection blog reader and the blog_posts reader into one store (storage
  stays single-source; the reader unifies at ONE URL `/blog/{id}` via origin-branching).
- Not deleting the carousel public read routes (carousel-origin posts still render through them, now
  reached via the single `/blog/{id}` branch; the FE fallback also relies on them).
- No new editorial/AI capabilities on the private surface beyond what exists.
- Public analytics / view-count (out-of-band, separate ticket). CDN cache + purge (owned follow-up).

## Resolved (was open) — from evidence check 2026-07-01

1. **Slug:** UNIQUE-enforced → `{id_or_slug}` may resolve by slug or id; v1 may ship slug.
2. **Design tokens:** default public theme for non-carousel posts; carousel-origin enriches via
   `origin`+`project_id`+`/design`. Never 404 for missing tokens.
3. **Homepage / dual-feed:** resolved by single canonical `/blog/{id}` (origin-branch) + FE fallback
   to the carousel feed on public-endpoint error. No duplicate URLs.
