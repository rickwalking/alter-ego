# AE-0003: Landing Page CSS Effects & Responsive Layout

## Status

Proposed (2026-06-02)

## Context

The marketing landing page was rebuilt in Next.js with inline styles matching the neon shell palette, but interaction and responsive behavior from the hardened static mockup (`frontend/public/redesign/index.html`) were not ported. Inline React styles cannot express `:hover`, `:active`, or media queries; the only responsive rule in `page.tsx` targets a non-existent `.container-posts` class.

Impeccable `harden` audit compared mockup vs live page and identified gaps limited to existing sections (hero, stats, features, posts, about).

## Decision

1. Introduce `landing.module.css` co-located with `(marketing)/page.tsx` for all interactive and breakpoint styles.
2. Port mockup hover/active and responsive rules verbatim where they apply to existing DOM.
3. Add hero glitch on line 1 only; keep gradient on Alter-Ego highlight per PRODUCT/DESIGN redlines.
4. Exclude mockup-only sections (theme showcase, CTA block).
5. Stats numbers: solid cyan + text-shadow (DESIGN.md) rather than gradient (unless product owner overrides).
