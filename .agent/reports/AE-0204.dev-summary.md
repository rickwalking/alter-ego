## Dual-write vs single-write decision

**Dual-write at the carousel-blog chokepoint (reversible mirror).** The writers
(`editorial_distribution_pack`, `refine_copy`, the publish endpoint) keep setting
the embedded `carousel_projects` columns, but the existing single chokepoint
`sync_carousel_blog_post` (which every carousel write already funnels through) now
ALSO mirrors caption/LinkedIn into `blog_posts.distribution` on the same row, in the
same transaction. The canonical home ŌĆö not the embedded column ŌĆö is the source of
truth every READER consumes (via the repository read overlay). The embedded columns
are retained but read-dead, so the AE-0205 drop is data-loss-free and this change is
fully reversible. The checkpoint sync is decoupled so a resumed `AsyncPostgresSaver`
checkpoint cannot resurrect an embedded write as a second source of truth.
