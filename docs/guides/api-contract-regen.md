# API Contract Regeneration (`make regen-contracts`)

Every change to the backend API contract (routes, request/response schemas,
status codes) must regenerate **four pinned artifacts**. Each is guarded by its
own CI gate, so a missed regen surfaces as a red gate — historically one red
gate at a time (AE-0325 / kaizen session-2026-07-22 FC-5). Run them all in one
command instead:

```bash
make regen-contracts        # or: bash scripts/dev/regen_contracts.sh
```

## The four artifacts

| # | Artifact | Regenerated by |
|---|----------|----------------|
| 1 | `docs/architecture/openapi.json` | `backend/scripts/export_openapi.py` |
| 2 | `backend/tests/snapshots/openapi_routes.json` | `REGEN_ROUTE_SNAPSHOT=1 pytest tests/unit/test_route_snapshot.py` |
| 3 | `backend/tests/snapshots/publishing/*.json` | `pytest tests/integration/test_publishing_safety_net.py --snapshot-update` |
| 4 | `backend/tests/snapshots/editorial/*.json` | `pytest tests/integration/test_carousel_workflow_safety_net.py --snapshot-update` |

## What the script guarantees

- **Fail-fast**: the first failing step aborts with a `regen INCOMPLETE` banner.
  Never commit after a partial run — the artifacts may disagree with each other.
- **Read-only verification**: after regenerating, it re-runs all four checks
  WITHOUT regen flags (`export_openapi.py --check` + the three snapshot tests)
  and only exits 0 when everything verifies green.
- **CWD-safe**: it `cd`s into `backend/` itself. (Running `export_openapi.py`
  from the repo root fails: the script loads `.env` from the current directory,
  and the ROOT `.env` carries an `ENCRYPTION_KEY` the backend `Settings`
  reject. Always run backend scripts from `backend/` — or use this wrapper.)

## When to run it

- After adding/removing/renaming a route or changing any Pydantic
  request/response schema.
- After a red `openapi`/route-snapshot/publishing/editorial snapshot gate that
  points at contract drift you intended.
- NOT to paper over unintended drift: if the diff surprises you, the code
  change is wrong, not the artifact. The pinned CI gates remain the enforcers —
  this script only removes the regen friction.

Related: `frontend/scripts/check-schema-drift.mjs` compares the frontend Zod
schemas against artifact #1; regenerate it before chasing frontend drift
reports.
